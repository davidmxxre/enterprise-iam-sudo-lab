## Purpose
Introduce the enterprise sudo lab and its objectives

## Scope
Covers user role definitions, sudo configuration, logging, SELinux integration, and recovery.

## Environment
- RHEL 9.5
- Local users/groups ('linux_admins', 'ap_opps', 'sec_audit')
- Services: 'httpd' for application operators

## Objectives
- Implement IAM model with least privilege
- Enforce command restrictions
- Enable forensic logging and auditing
- Test failure recovery and misconfigurations

## Key Takeaways
The lab demonstrates how sudo and system controls enforce least prvilege while providing visibility and accountability.
